VS Toolkit, the mobile HTML5 toolkit used with VSD.

This toolkit was created to easily develop Hybrid Applications for mobile devices (for instance on top of PhoneGap)

The main features are:

  • Objects oriented toolkit
  • Framework driven by a component approach
  • Optimized for mobile devices
    • Small memory footprint and memory deallocation
    • Performance (JS and Rendering)

The IDE ViniSketch Designer (VSD) use this toolkit for creating application.

Object Model

Create a class 

var myClass = vs.core.createClass ({

  // parent class
  parent: vs.ui.View,

  // properties declaration
  properties : {text: vs.core.Object.PROPERTY_IN_OUT},

  // prototype declaration
  aMethod : function (p1, p2, …) {
    this._super (p1, p2, …);
    …
  }, 
  …
});

Create an object from a class 

var config = {
  id: 'xxx',
  text: ''
}

// Object construction
var obj = new myClass (config);

// Object initialization
obj.init ();

// property manipulation
obj.text = 'Hello';

Properties

Basic declaration 

  properties : {propName: vs.core.Object.PROPERTY_IN_OUT},

Possible export values are:

  • vs.core.Object.PROPERTY_IN_OUT
  • vs.core.Object.PROPERTY_IN
  • vs.core.Object.PROPERTY_OUT

Descriptor declaration 

properties : {
  propName: {
    set : function (v) {
      …
      this._prop_name = v;
    },

    get : function () {
      …
      return this._prop_name
    }
  }
},

Path declaration

This third syntax allows to easily export a child component's property.

Within this example, my class exports as own property, the property named 'text' of its child 'textField'. 

properties : {text: "textField#text"}

This declaration is similar to: 

properties : {
  text: {
    set : function (v) {
      this.textField.text = v;
    },

    get : function () {
      return this.textField.text;
    }
  }
},

Create your first application

JavaScript bootstrap 

var myApp = vs.core.createClass ({

  /** parent class */
  parent: vs.ui.Application,

  /** called when vs.ui.Application.start (); is called */
  applicationStarted : function (event) {
    …
  }
});

function loadApplication () {
  new myApp ({id:"_myApp_"}).init ();

  vs.ui.Application.start ();
}

<html id="_myApp_">

Template

WARNING: The API is not yet fully committed.

Template is a simple way for creating view of a component.
A template is a HTML text fragment containing template tags. There is two ways to use template :

  • By expanding tags using values provides in an Object
  • By generating a vs.ui.View with properties linked to tags

Typical template description: 

var str = '<span style="${style}">name:${lastname},${firstname}</span>';

1- Expanding the template: 

var myTemplate = new Template (str);

var values = {
  lastname : "Doo",
  firstname : "John",
  style : "color:blue"
};

console.log (myTemplate.apply (values));
// -> <span style="color:blue">name:Doo,John</span>

2- Generating a vs.ui.View from the template: 

var myTemplate = new Template (str);

var myView = myTemplate.compileView ();

myApp.add (myView);

// property changes, automatically update the DOM
myView.lastname = "Doo";
myView.firstname = "John";
myView.style = "color:blue";
